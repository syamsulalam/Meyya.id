# Bug Fix Report: Clerk Auth Integration

Berikut adalah analisis bug, potensi masalah, dan perbaikan berdasarkan panduan terbaru pada `CLERK_SETUP.md`. Tombol daftar dan Google Login tidak berfungsi karena aplikasi kehilangan `ClerkProvider` yang wajib ada untuk menjalankan fungsi dari hook autentikasi Clerk.

| Bug | Alasan Potensi Bug / Dampak | Fix / Solusi yang Diterapkan |
| --- | --- | --- |
| **`ClerkProvider` absen dari Root Layout / `main.tsx`** | Ini adalah penyebab utama (root cause) mengapa tombol Daftar dan Google Login tidak berfungsi. Hook seperti `useSignIn` dan `useSignUp` membutuhkan context internal dari `<ClerkProvider>`. Jika tidak ada, proses auth tidak tereksekusi secara fatal. | Membungkus komponen `<App />` di dalam `src/main.tsx` dengan `<ClerkProvider afterSignOutUrl="/">`. |
| **Error Timeout Telemetry (`clerk-telemetry.com`)** | Beberapa browser (seperti Brave dengan Shields) atau ekstensi Ad-Blocker / Tracker-Blocker seringkali memblokir endpoint tracking telemetry dari Clerk yang menyebabkan _Failed to load resource: net::ERR_CONNECTION_TIMED_OUT_. Hal ini umumnya tidak mengganggu inti login/register, namun dapat menyebabkan beberapa log error di console terkait auto-fill. | Mengatur `telemetry={false}` di `<ClerkProvider>` dan menambahkan script untuk mengabaikan auto-fill `data-1p-ignore="true"` pada beberapa field formulir untuk meminimalisasi crash auto-fill overlay Clerk. |
| **Koneksi Clerk Terblokir / Key Kosong** | Jika pengguna belum memasukan rahasia (Secret / Publishable Key), maka Clerk menggunakan fallback test-key dari library, yang mungkin sudah expired atau domain-nya ter-blacklist. Hal ini membuat isLoaded selalu ngestuck di false dan fungsi auth akan menampilkan error "Koneksi ke Clerk belum siap". | Menambahkan fungsi pengecekan `checkClerkEnv` di `Auth.tsx`. Jika variabel lingkungan `VITE_CLERK_PUBLISHABLE_KEY` belum terpasang atau salah format, sistem akan memunculkan banner *Peringatan Konfigurasi* di UI tanpa menghalangi form agar developer/user tahu akar masalahnya. |
| **Error Message tidak terlihat karena form Registration panjang** | Ketika proses submit gagal atau ada error pada kredensial, pengguna tidak menyadarinya karena pesan error muncul di atas form sementara pengguna ada di bawah page. | Menambahkan `useRef` dan `scrollIntoView` pada box error message di `Auth.tsx`. Form akan otomatis bergeser (scroll) ke letak log / warning agar terlihat jelas oleh user. |
